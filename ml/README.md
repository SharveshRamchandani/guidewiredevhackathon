# GigShield ML Service

Internal ML microservice for GigShield — a parametric insurance platform for food delivery gig workers in India (Swiggy/Zomato). Built for Guidewire DEVTrails 2026.

This service handles two things: scoring a worker's insurance risk and detecting fraud on claims. It also exposes weather, AQI, and alert trigger endpoints that the backend uses to decide whether to auto-trigger a claim.

No auth on any endpoint — this service is internal only.

---

## What it does

| Endpoint | What it returns |
|---|---|
| `POST /ml/risk-score` | Risk score + suggested weekly premium for a worker |
| `POST /ml/fraud-score` | Fraud probability + auto-approve / review / reject decision |
| `GET /triggers/weather` | Live weather data + whether the threshold is breached for a zone |
| `GET /triggers/aqi` | Live AQI + breach status for a zone |
| `GET /triggers/mock-alerts` | Strike / curfew / traffic alerts for a zone |
| `GET /health` | Basic health check |

## Running locally

```bash
# Set up virtual environment
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS/Linux

pip install -r requirements.txt
```

Copy `.env` and fill in your API keys:

```env
OPENWEATHERMAP_API_KEY=your-key
AQICN_API_KEY=your-key
```

Generate training data and train the models:

```bash
python models/generate_data.py
python models/train_risk.py
python models/train_fraud.py
```

Start the server:

```bash
uvicorn main:app --reload --port 8000
```

Swagger docs at `http://localhost:8000/docs`.

## API

### POST /ml/risk-score

Scores a worker's risk level based on their activity profile and zone. Used by the backend to calculate their weekly premium.

Request:
```json
{
  "worker_id": "W_0012",
  "zone_id": 3,
  "platform": "Swiggy",
  "months_active": 14,
  "avg_daily_hours": 8.5,
  "past_claims_count": 1,
  "zone_flood_risk": 0.75,
  "zone_heat_risk": 0.4
}
```

Response:
```json
{
  "risk_score": 0.62,
  "risk_label": "medium",
  "suggested_premium": 58.40,
  "coverage_recommended": 600
}
```

`risk_label` is one of `low`, `medium`, or `high`. `suggested_premium` is in INR per week.

---

### POST /ml/fraud-score

Scores a submitted claim for fraud. The backend uses the decision field to auto-approve, queue for manual review, or reject outright.

Request:
```json
{
  "claim_id": "CLM-1847",
  "worker_id": "W_0012",
  "gps_zone_match": true,
  "claim_velocity_7d": 2,
  "historical_zone_presence": 0.85,
  "time_since_event_seconds": 1800,
  "platform_activity_during_event": 0.7
}
```

Response:
```json
{
  "fraud_score": 0.15,
  "decision": "auto_approve",
  "reason": "Low velocity, GPS zone match confirmed"
}
```

Decision thresholds:

| fraud_score | decision |
|---|---|
| < 0.30 | auto_approve |
| 0.30 – 0.60 | manual_review |
| > 0.60 | auto_reject |

### GET /triggers/weather?zone_id={id}

Returns weather readings for a zone and whether they cross a trigger threshold.

```json
{
  "zone_id": 3,
  "temperature_celsius": 38.5,
  "rainfall_mm": 0.0,
  "threshold_breached": false,
  "severity": 0.0,
  "trigger_type": "none"
}
```

Trigger rules:
- `rainfall_mm > 100` → `flood`, severity 1.0
- `rainfall_mm > 50` → `heavy_rain`, severity 0.7
- `temperature_celsius > 45` → `extreme_heat`, severity 0.5

## Models

**Risk — Random Forest Classifier**

Trained on synthetic data generated by `models/generate_data.py` (500 rows, seed 42). Features: `months_active`, `avg_daily_hours`, `past_claims_count`, `zone_flood_risk`, `zone_heat_risk`. Target is `risk_label` (low/medium/high).

**Fraud — Isolation Forest + Rules**

Unsupervised anomaly detection. Learns what normal claim patterns look like and flags outliers. Rule-based overrides handle high-confidence cases (GPS mismatch, very high claim velocity). Features: `gps_zone_match`, `claim_velocity_7d`, `historical_zone_presence`, `time_since_event_seconds`, `platform_activity_during_event`.

Both models are saved with `joblib` to `models/saved/`.
